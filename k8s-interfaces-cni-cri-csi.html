CNI
Container Network Interface
Defines how pods get network connectivity
CNI is a specification and set of libraries for configuring network interfaces in Linux containers. When a pod is created, kubelet calls the CNI plugin to assign an IP address and set up routing rules.
Popular Plugins
Calico
Flannel
Cilium
Weave
What CNI does
1
Assigns a unique IP address to each pod from the cluster CIDR
2
Creates virtual network interfaces (veth pairs) in pod namespaces
3
Configures routes so pods can communicate across nodes
4
Enforces NetworkPolicy rules via eBPF or iptables
CRI
Container Runtime Interface
Abstracts the container runtime layer
CRI is a plugin interface that enables kubelet to use different container runtimes without recompilation. It uses a gRPC API to manage pod sandboxes and container lifecycles.
Supported Runtimes
containerd
CRI-O
Docker*
gVisor
CRI gRPC Calls
1
PullImage — fetch image from registry to local store
2
RunPodSandbox — create network/IPC namespaces (pause container)
3
CreateContainer — init the container in the sandbox
4
StartContainer — begin execution of the container process
CSI
Container Storage Interface
Standard API for storage provisioning
CSI defines a standard interface between container orchestrators and storage systems, allowing storage vendors to develop a single plugin that works across Kubernetes and other orchestrators.
Storage Drivers
AWS EBS
GCE PD
Azure Disk
Rook/Ceph
Volume Lifecycle
1
CreateVolume — provision storage on the backend (PVC → PV)
2
ControllerPublish — attach volume to the target node
3
NodeStageVolume — format & mount at a global staging path
4
NodePublishVolume — bind-mount into pod's container path
