Kubernetes
Orchestration

How Kubernetes automatically deploys, scales, heals, and manages containerised workloads across a cluster of machines.

Control Plane
The brain of Kubernetes — runs on master nodes
kube-apiserver
The single entry point — all kubectl, controller, and kubelet calls go through here
Validates & persists all state to etcd
Enforces RBAC, admission webhooks
Watch API notifies controllers of changes
Horizontally scalable
etcd
Distributed key-value store — the source of truth for all cluster state
Stores Pods, Services, ConfigMaps, Secrets…
Uses Raft consensus protocol
Strongly consistent & highly available
Only API server reads/writes etcd
kube-scheduler
Assigns unscheduled pods to appropriate worker nodes
Filtering: removes nodes that don't meet requirements
Scoring: ranks feasible nodes (affinity, taints)
Respects resource requests, limits, topology
Pluggable via Scheduling Framework
controller-manager
Runs control loops that continuously reconcile desired vs actual state
Deployment, ReplicaSet, DaemonSet controllers
Job, CronJob, StatefulSet controllers
Node, Namespace, ServiceAccount controllers
Each loop watches etcd via API server

Workload Resource Types

Kubernetes abstractions for different application patterns
📦
Pod
Smallest unit. One or more containers sharing network & storage
🔄
ReplicaSet
Maintains N replicas of a pod. Replaced by Deployment
🚀
Deployment
Stateless apps. Rolling updates, rollbacks, scaling
🗄️
StatefulSet
Ordered pods with stable IDs & persistent volumes
🌐
DaemonSet
Runs one pod per node (logging, monitoring, CNI)
⏱️
CronJob
Scheduled batch workloads using cron expressions

The Reconciliation Loop

Every Kubernetes controller runs an infinite control loop: observe, compare, act. This is how K8s self-heals.

👁️
Observe Watch API — controller receives events when resources change
🔍
Diff Compare desired state (spec) with current state (status)
⚡
Act Create / update / delete resources to drive toward desired state
✅
Update Status Write status.conditions back to API server, then loop again

Self-Healing in Action

When something goes wrong, Kubernetes automatically detects and corrects the deviation from desired state.

💥
Pod Crashes Process exits with non-zero code. kubelet detects immediately
🔄
Restart Policy Applied Always / OnFailure restarts pod with exponential back-off
📊
ReplicaSet Reconciles Controller sees actual < desired; creates a replacement pod
✅
Traffic Restored readinessProbe passes; Endpoints updated; Service routes traffic
